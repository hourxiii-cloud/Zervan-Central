# Doctrine — Entry Point

**Canonical entry point:** `CATHEDRAL.md`

All doctrine in this directory is subordinate to Cathedral and must remain consistent with it.

## Reading Order (recommended)

1. `CATHEDRAL.md` — root governance / admissibility
2. `ORABORUS.md` — recursion governance (under Cathedral)
3. `PMC.md` + `PMC_ARENA.md` — paradox management / consistency enforcement
4. `MC.md` — meta collapse (response admissibility gate; consumes PMC output only)
5. `ZSD.md` — stabilization doctrine
6. `ZEA.md` — execution architecture
7. `ZEE.md` — evaluation envelope
8. `PMC_MC_INTERFACE.md` — canonical PMC → MC interface contract (binding boundary)

## Push-Down Rule

No doctrine file may define authority above Cathedral.  
If conflict exists, Cathedral arbitrates.

## Non-Bypassable Gate

No response, escalation, or governance artifact may be produced without passing
through **MC** as the response admissibility layer.

MC does not analyze raw data, infer intent, or retain memory.  
MC consumes only PMC outputs, explicit uncertainty annotations, and known governance constraints.
