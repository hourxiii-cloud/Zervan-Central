# Artifact-Class Operator Eligibility Matrix  
**Version:** 1.0.0  
**Status:** CANONICAL  
**Layer:** TOP DOMAIN • GOVERNANCE • EPISTEMIC CONTROL  

---

## 0. Purpose (Non-Negotiable)

This matrix defines **which Zervan operator families are epistemically eligible to execute** against a given **artifact class**.

It exists to prevent:
- semantic overreach  
- projection masquerading as insight  
- safe-but-unjustified interpretation  
- collapse inflation on low-stakes artifacts  

This matrix governs **execution eligibility**, not output censorship.  
**Guardrails remain downstream and orthogonal.**

---

## 1. Core Principle

> **Not all signals deserve all lenses.**

Operator execution is **conditional on artifact class**, not on:
- user intent  
- emotional charge  
- perceived profundity  
- rhetorical intensity  

Eligibility is determined **before analysis begins**.

---

## 2. Artifact Classes (v1 Canon)

Artifact classes are determined by **form, scale, density, and stake**, not ideology.

### A. SHORT-FORM COMEDY / PERFORMANCE
Examples:
- Stand-up bits  
- Shorts / Reels / TikTok clips  
- Satirical monologues  
- Provocation-as-play  

Characteristics:
- Low duration  
- High rhetorical play  
- Simulated stakes  
- Intentional inversion / exaggeration  

---

### B. LONG-FORM CULTURAL / PHILOSOPHICAL WORK
Examples:
- Essays  
- Manifestos  
- Lectures  
- Interviews (full context)  

Characteristics:
- Sustained argument  
- Explicit claims  
- Recursive development  
- Medium stakes  

---

### C. TECHNICAL / SCIENTIFIC / CONTRACTUAL
Examples:
- Specifications  
- Research papers  
- Schemas  
- Legal text  

Characteristics:
- Precision required  
- Determinism expected  
- High falsifiability  

---

### D. TESTIMONY / ADVERSARIAL / THREAT ARTIFACT
Examples:
- Whistleblower statements  
- Threat communications  
- Legal testimony  
- Incident reports  

Characteristics:
- Real stakes  
- Power asymmetry  
- Potential harm vectors  

---

## 3. Operator Families

Operators are grouped into families:

- **SPIN** — inversion, reference-frame shifts  
- **COUPLING** — dyadic / relational binding  
- **DIVERGENCE** — wobble, instability detection  
- **RECURSION** — self-reference, bounded loops  
- **STABILIZATION (ZSD)** — tension → resolution  
- **ECP** — external coherence  
- **THREAT / ADVERSARIAL** — dominance, coercion, risk  
- **COLLAPSE PHYSICS** — attractors, inevitability, force  
- **MEANING MAXIMIZER** — latent truth extraction  
- **INTEGRITY / HASH** — provenance, tamper validation
